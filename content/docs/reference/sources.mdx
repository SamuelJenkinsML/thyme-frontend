A source ingests events from an external system into a Thyme dataset. See [Sources](/docs/concepts/sources) for a conceptual overview; this page is the connector-by-connector parameter reference.

## `@source` decorator

```python
@source(
    connector,
    cursor="ts",
    every="1m",
    disorder="5m",
    cdc="append",
)
@dataset(index=True)
class Transaction: ...
```

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `connector` | connector object | required | The data source to read from |
| `cursor` | `str` | `""` | Field used for incremental reads (polling only) |
| `every` | `str` | `""` | Poll interval: `"1m"`, `"5m"`, `"1h"` (polling only) |
| `disorder` | `str` | `""` | Maximum late-arrival tolerance: `"5m"`, `"1h"`, `"1d"` |
| `cdc` | `str` | `"append"` | Change data capture mode — see below |

Streaming sources consume continuously and ignore `every` / `cursor`.

---

## CDC modes

| Mode | Row semantics | Use for |
|------|---------------|---------|
| `append` | Each row is a new event | Event logs, transactions, clicks |
| `upsert` | Each row is the latest state for its key | User profiles, inventory snapshots |
| `debezium` | Row is a Debezium envelope (`before`, `after`, `op`) | Postgres/MySQL CDC via Debezium |

`debezium` mode respects `INSERT`, `UPDATE`, and `DELETE` operations from the envelope.

## Streaming connectors

### `KafkaSource`

Consumes from an Apache Kafka topic.

```python
from thyme.connectors import KafkaSource

KafkaSource(
    brokers="kafka-1:9092,kafka-2:9092",
    topic="transactions",
    group_id="thyme-transactions",
)
```

| Parameter | Type | Description |
|-----------|------|-------------|
| `brokers` | `str` | Comma-separated broker addresses |
| `topic` | `str` | Topic name to consume from |
| `group_id` | `str` | Consumer group ID |

### `KinesisSource`

Consumes from an AWS Kinesis stream.

```python
from thyme.connectors import KinesisSource

KinesisSource(
    stream="user-events",
    region="us-east-1",
    starting_position="LATEST",
)
```

| Parameter | Type | Description |
|-----------|------|-------------|
| `stream` | `str` | Kinesis stream name |
| `region` | `str` | AWS region |
| `starting_position` | `str` | `LATEST` or `TRIM_HORIZON` |

## Polling connectors

### `IcebergSource`

Reads from an Apache Iceberg table.

```python
from thyme.connectors import IcebergSource

IcebergSource(
    catalog="prod",
    database="events",
    table="transactions",
)
```

| Parameter | Type | Description |
|-----------|------|-------------|
| `catalog` | `str` | Iceberg catalog name |
| `database` | `str` | Database or namespace |
| `table` | `str` | Table name |

### `PostgresSource`

Reads from a Postgres table incrementally via the cursor field.

```python
from thyme.connectors import PostgresSource

PostgresSource(
    dsn="postgresql://user:pass@host:5432/db",
    table="transactions",
)
```

| Parameter | Type | Description |
|-----------|------|-------------|
| `dsn` | `str` | Postgres connection string |
| `table` | `str` | Table name |

For logical-replication CDC rather than cursor polling, run Debezium upstream and consume with `KafkaSource` + `cdc="debezium"`.

### `S3Source`

Reads partitioned files from an S3 prefix.

```python
from thyme.connectors import S3Source

S3Source(
    bucket="events-prod",
    prefix="transactions/",
    format="parquet",
)
```

| Parameter | Type | Description |
|-----------|------|-------------|
| `bucket` | `str` | S3 bucket name |
| `prefix` | `str` | Key prefix to scan |
| `format` | `str` | `parquet`, `json`, or `csv` |

### `BigQuerySource`

Reads from a BigQuery table.

```python
from thyme.connectors import BigQuerySource

BigQuerySource(
    project="my-project",
    dataset="events",
    table="transactions",
)
```

### `SnowflakeSource`

Reads from a Snowflake table.

```python
from thyme.connectors import SnowflakeSource

SnowflakeSource(
    account="my-account",
    warehouse="COMPUTE_WH",
    database="EVENTS",
    schema="PUBLIC",
    table="TRANSACTIONS",
)
```
